# Сквозное шифрование с OLM

**Languages:** [English](OLM.md) | [Русский](OLM.ru.md)

# Что такое OLM?

**OLM это криптографический протокол**, который позволяет двум людям безопасно общаться через недоверенную сеть. Он сочетает эллиптическую криптографию на Curve25519 с алгоритмом Double Ratchet, обеспечивая шифрование, которое защищает прошлые сообщения даже в случае компрометации текущих ключей и не позволяет подделывать будущие сообщения, даже если система уже скомпрометирована.

# Установление сеанса

### Генерация ключей

Перед началом обмена сообщениями каждая сторона генерирует два ключа:

- **Идентификационный ключ** - постоянный идентификатор пользователя, действующий на протяжении всей сессии.
- **Одноразовый ключ** - ключ для однократного использования при первичной настройке, действующий только на этапе установления соединения.

Сторона A генерирует оба ключа, объединяет их в одну строку и передаёт этот набор стороне B вне канала связи (например, через терминал, по электронной почте и т.п.).

### Установление сеанса

**Если вы начинаете переписку (исходящий сеанс):**
1. Вы получаете набор ключей стороны B.
2. Выполняете математическое согласование ключей (ECDH) между своими закрытыми ключами и их открытыми ключами.
3. В результате получается общий секрет, который могут вывести только вы и они.
4. Вы создаёте PreKeyMessage - своё первое зашифрованное сообщение, содержащее дополнительную информацию, чтобы другая сторона могла синхронизироваться.
5. Вы отправляете им это PreKeyMessage.

**Если переписку начинают они, а вы получаете сообщение первыми (входящий сеанс):**
1. Вы получаете набор ключей стороны A.
2. Вы получаете их PreKeyMessage.
3. Извлекаете их идентификационные данные и дополнительную информацию.
4. Выполняете те же вычисления для согласования ключей.
5. Получаете тот же самый общий секрет.

После этого обе стороны имеют **синхронизированное состояние сеанса**.

# Double Ratchet

После установления сеанса сообщения не используют статические ключи. Вместо этого **механизм ratchet** вычисляет новый ключ для каждого сообщения.

Каждое сообщение использует уникальный ключ, выведенный из текущего состояния сеанса. После шифрования состояние сеанса обновляется, поэтому для следующего сообщения используется уже другой ключ. Старые ключи удаляются. Восстановить их невозможно, даже если текущий ключ сеанса будет раскрыт.

# Типы сообщений

Протокол использует два формата сообщений:

- **PreKeyMessage**: отправляется только в первом сообщении инициатором. Содержит идентификационные данные, материал для согласования ключей и зашифрованное содержимое.
- **Message**: используется для всех последующих сообщений. Содержит только зашифрованное содержимое и информацию о цепочке ключей. Он значительно меньше.

# Кодирование Base64

Ключи и зашифрованные сообщения - это двоичные данные. Чтобы передавать их как текст (например, вставлять в терминал или копировать через чат), их кодируют в Base64 - тексто-безопасный формат, использующий буквы, цифры и символы. Получатель декодирует Base64 обратно в двоичный вид перед использованием криптографического материала.

# Управление сеансами

Код управляет **несколькими независимыми зашифрованными каналами** с разными контактами. Для каждого контакта создаётся свой сеанс со своим состоянием ratchet. Вы выбираете, с каким контактом переписываетесь, а затем шифруете и расшифровываете сообщения - система автоматически выполняет продвижение ratchet.

# Полный пример установления сеанса

**Сторона A хочет написать стороне B:**

1. Сторона A генерирует свои ключи и передаёт их.
2. Сторона B копирует ключи стороны A и генерирует свои.
3. Сторона B передаёт свои ключи.
4. Сторона A копирует ключи стороны B и устанавливает исходящий сеанс.
5. Сторона A шифрует первое сообщение (PreKeyMessage) и отображает его.
6. Сторона B получает ключи стороны A и зашифрованное сообщение, устанавливает входящий сеанс и расшифровывает его.
7. После этого у обеих сторон синхронизировано состояние сеанса, и они могут свободно обмениваться зашифрованными сообщениями.
8. Каждое новое сообщение независимо продвигает оба ratchet в одном и том же направлении.

# Гарантии безопасности

**Конфиденциальность**: Сообщения шифруются с помощью AES-256-GCM, поэтому их может прочитать только адресат.

**Аутентичность**: Каждое сообщение содержит криптографическое подтверждение того, что оно не было подделано или изменено.

**Прямая секретность**: Еесли ключ сеанса будет раскрыт сегодня, злоумышленники всё равно не смогут прочитать прошлые сообщения, потому что старые ключи удалены.

**Восстановление после компрометации**: Если система будет взломана в середине переписки, злоумышленники не смогут подделывать ваши будущие сообщения, потому что ключи двигаются только вперёд.

**Защита от повторного воспроизведения**: Одно и то же сообщение нельзя расшифровать дважды, ratchet предотвращает повторное использование.
