# Сквозное шифрование с OLM

**Languages:** [English](OLM.md) | [Русский](OLM.ru.md)

# Что такое OLM?

OLM - это криптографический протокол для безопасной связи через недоверенную сеть. Он сочетает обмен ключами Curve25519 с алгоритмом Double Ratchet, обеспечивая сквозное шифрование и совершенную прямую секретность.

На практике это означает, что сообщения шифруются локально, старые ключи сообщений удаляются, а будущие сообщения остаются защищёнными даже при частичной компрометации сессии.

# Установление сеанса

### Материал ключей

Перед началом обмена сообщениями каждая сторона готовит ключевой материал для установления сеанса:

- **Идентифицирующий ключ** - Долгоживущий ключ, который используется для идентификации пользователя в течение сеанса.
- **Одноразовый ключ** - Ключ, используемый один раз при первоначальной настройке сеанса.

### Установление сеанса

**Если сеанс инициируете вы:**
1. Вы получаете набор ключей другой стороны
2. Выполняете обмен ключами ECDH, используя свои закрытые ключи и их открытые ключи
3. В результате получается общий секрет, который могут вычислить только обе стороны
4. Вы создаёте первое зашифрованное сообщение, которое содержит дополнительный ключевой материал для синхронизации сеанса
5. Вы отправляете это сообщение другой стороне

**Если первым приходит сообщение от другой стороны:**
1. Вы получаете набор ключей другой стороны
2. Вы получаете их первое зашифрованное сообщение
3. Извлекаете из сообщения ключевой материал
4. Выполняете те же шаги обмена ключами
5. Получаете тот же общий секрет и синхронизируете состояние сеанса

После этого обе стороны получают общий зашифрованный сеанс.

# Double Ratchet

После установления сеанса сообщения больше не используют статические ключи. Вместо этого сеанс продвигается с помощью ratchet-механизма, который выводит новые ключи для каждого сообщения.

Каждое сообщение использует уникальный ключ, полученный из текущего состояния сеанса. После шифрования или расшифрования сеанс продвигается дальше, а старые ключи сообщений удаляются, что снижает ущерб при компрометации ключей.

# Типы сообщений

Протокол использует два основных формата сообщений:

- **PreKeyMessage** - Используется для первого сообщения в сеансе и содержит начальный ключевой материал и зашифрованное содержимое.
- **Message** -Ииспользуется для всех последующих сообщений и содержит только зашифрованное содержимое и данные ratchet-механизма.

# Кодирование Base64

Ключи и зашифрованные сообщения - это бинарные данные. Чтобы их было удобно копировать и вставлять через текстовые каналы, они кодируются в Base64. Получатель декодирует их обратно в бинарный формат перед использованием.

# Управление сеансами

Приложение управляет несколькими независимыми зашифрованными сеансами, по одному на контакт. У каждого контакта своё состояние ratchet-механизма, поэтому диалоги изолированы друг от друга.

Вы выбираете контакт, затем шифруете или расшифровываете сообщения для этого сеанса. Приложение автоматически управляет продвижением ratchet-механизма.

# Полный пример установления сеанса

1. Сторона A генерирует ключевой материал и передаёт его
2. Сторона B получает его и готовит свои ключи
3. Сторона B отправляет свой набор ключей обратно
4. Сторона A устанавливает сеанс и отправляет первое зашифрованное сообщение
5. Сторона B получает набор ключей и сообщение, синхронизирует сеанс и расшифровывает его
6. После этого обе стороны используют одно и то же состояние сеанса
7. Каждое новое сообщение независимо продвигает ratchet-механизм на обеих сторонах

# Гарантии безопасности

- **Конфиденциальность** - Сообщения зашифрованы так, что их может прочитать только адресат
- **Аутентичность** - Подмена или изменение сообщений обнаруживается
- **Совершенная прямая секретность** - Прошлые сообщения остаются защищёнными даже при компрометации текущих ключей
- **Защита после компрометации** - Если сеанс был скомпрометирован, будущие сообщения остаются защищёнными после продвижения ratchet-механизма
- **Защита от повторного воспроизведения** - Состояние ratchet-механизма ограничивает повторное использование старого криптоматериала
