# Обзор

OpenE2E - это ручной безопасный мессенджер для обмена зашифрованными сообщениями через любой канал: от SMS и email до мессенджеров, соцсетей и других недоверенных каналов.

Приложение локально шифрует и расшифровывает сообщения, а между людьми передаётся уже зашифрованный текст любым доступным способом. Это делает проект полезным там, где нет удобного безопасного канала связи.

OpenE2E использует **Matrix OLM** через библиотеку `voldozemac`, **AES-256-GCM** для шифрования сообщений и **сквозное шифровние с совершенной прямой секретностью (PFS)**.

Подробнее о модели шифрования и внутреннем устройстве: [OLM.ru.md](OLM.ru.md)

# Возможности

- **Ручной безопасный чат** - Обмен зашифрованными сообщениями через копирование с вставкой и любой канал
- **Сквозное шифрование** - Сообщения шифруются локально и доступны только адресату
- **Совершенная прямая секретность** - Прошлые сообщения остаются защищёнными даже при компрометации будущих ключей
- **Работа через любой канал** - SMS, email, мессенджеры, гос. платформы и другие публичные каналы
- **Локальное хранение** - Все данные остаются на вашем устройстве
- **Зашифрованное хранение** - Сообщения и сессии хранятся локально в зашифрованной базе данных [В разработке]
- **Chat-like интерфейс** - Удобный UI на **Slint** [В разработке]
- **Написан на Rust** - Безопасно и производительно

# Как это работает

1. **Создайте сессию** - Начните новый диалог в приложении
2. **Обменяйтесь публичными ключами** - Сгенерируйте эфимерный публичный ключ и передайте его контакту любым каналом
3. **Получите ключ собеседника** - Аставьте его публичный ключ в приложение, чтобы установить сессию
4. **Напишите сообщение** - Аведите текст в приложении
5. **Зашифруйте и скопируйте** - Приложение шифрует сообщение локально и выдаёт шифр для отправки
6. **Получите шифр** - Вставьте зашифрованное сообщение от собеседника в приложение
7. **Расшифруйте локально** - Приложение расшифровывает сообщение на вашем устройстве и показывает его в удобном виде
8. **Продолжайте диалог** - Повторяйте процесс для дальнейшего общения

# Безопасность

- **Совершенная прямая секретность (PFS)** - Алгоритм Double Ratchet в OLM обеспечивает сохранность прошлых сообщений даже при компрометации долгосрочных ключей
- **Сквозное шифрование** - Содержимое сообщений видно только двум конечным точкам
- **Только локальное хранение** - Нет облачной синхронизации и серверного хранения сообщений
- **Ручной обмен ключами** - Без автоматических допущений о доверии
- **Независимость от канала** - Зашифрованные данные можно передавать практически через любой канал

### Защита локальных данных

Сообщения и сессии хранятся локально в RocksDB. Шифрование хранилища **[Work in progress]**.
