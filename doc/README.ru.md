# Обзор

OpenE2E - это простой чат-подобный интерфейс с **сквозным шифрованием (E2E)** и **Perfect Forward Secrecy (PFS)** на базе **OLM** из **Matrix**, позволяющий безопасно общаться через небезопасные каналы связи.

Больше информации о выбранном алгоритме шифрования и его принципах работы: [OLM.ru.md](doc/OLM.ru.md)

# Возможности

- **Сквозное шифрование** - Все сообщения шифруются с использованием асимметричной криптографии и PFS
- **Простой обмен ключами** - Без сложной настройки. Просто сгенерируйте временные открытые ключи, передайте их любым каналом и установите защищённый сеанс
- **Чат-подобный интерфейс** - Удобный мессенджерный UI на **Slint**
- **Локальное хранение сообщений** - Все сообщения хранятся локально в читаемом формате чата
- **Режим маскировки** - При желании сообщения можно сохранять как медиафайлы, чтобы избежать ML-анализа и подозрительных шаблонов
- **Работа через недоверенные каналы** - Отправляйте зашифрованные сообщения через государственные мессенджеры, email, SMS или любую публичную платформу
- **Написано на Rust** - Безопасно для памяти и очень быстро

# Как это работает

1. **Создайте сеанс** - Начните новый разговор в приложении
2. **Обменяйтесь ключами** - Сгенерируйте и скопируйте свой временный открытый ключ и передайте его собеседнику
3. **Получите ключ** - Вставьте открытый ключ собеседника в приложение, чтобы установить защищённый сеанс
4. **Отправляйте сообщения** - Введите сообщение, скопируйте зашифрованный вывод и отправьте его через любой мессенджер
5. **Получайте сообщения** - Вставляйте полученные зашифрованные сообщения в приложение, чтобы расшифровать и прочитать их
6. **Просматривайте историю** - Все расшифрованные сообщения отображаются в удобном интерфейсе чата

# Функции безопасности

- **Perfect Forward Secrecy (PFS)** - Алгоритм Double Ratchet в OLM обеспечивает сохранность прошлых сообщений даже при компрометации долгосрочных ключей
- **Асимметричное шифрование** - Криптография с открытым ключом защищает от перехвата
- **Только локальное хранение** - Без облачной синхронизации, все данные остаются на вашем устройстве
- **Устойчивость к недоверенным каналам** - Сообщения можно отправлять через любую публичную платформу без дополнительной настройки

### Режим маскировки

Включите маскировку, чтобы хранить сообщения как медиафайлы вместо обычного текста:
- Предотвращает автоматический анализ структуры сообщений
- Сохраняет правдоподобное отрицание
- Доступ к сообщениям возможен только через приложение

# Ограничения

- Требуется ручной обмен ключами (без автоматического распределения ключей)
- Сообщения нужно вручную копировать и вставлять между каналами
- Нет поддержки нескольких устройств
